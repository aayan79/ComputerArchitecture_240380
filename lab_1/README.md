## Theory

### What is VHDL?

VHDL stands for **VHSIC Hardware Description Language** (VHSIC = Very High Speed Integrated Circuits). It is an IEEE industry-standard language used to model digital systems at many levels of abstraction — from the algorithmic level down to the gate level. Unlike conventional programming languages, VHDL describes how signals flow through logic gates and flip-flops **concurrently** (at the same time), not sequentially.

---

### VHDL Structure

A VHDL design is composed of three essential parts:

| Part | Description |
|------|-------------|
| **Library and Packages** | Contain predefined data types, constants, and functions. The most common is the `IEEE` library. |
| **Entity** | Defines the external interface (input/output ports) of the circuit — the "black box." |
| **Architecture** | Defines the internal logic or behavior of the entity — what is inside the black box. |

---

### Libraries and Packages

- **Library `std`:** Automatically included in every VHDL design. Provides basic data types such as `bit`, `integer`, `boolean`, and `character`.
- **Library `IEEE`:** The most widely used library. Provides `std_logic` and `std_logic_vector` types essential for real hardware design.

```vhdl
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;  -- Provides std_logic types
use IEEE.NUMERIC_STD.ALL;     -- Provides arithmetic on std_logic
```

---

### Entity

The entity declares the external ports of a circuit — its names, directions, and data types.

**Port Directions:**
- `in` — signal flows into the entity (input)
- `out` — signal flows out of the entity (output)
- `inout` — signal flows both ways (bidirectional)

---

### Architecture

The architecture defines the internal behavior of the entity. VHDL supports three architectural modeling styles:

| Style | Description |
|-------|-------------|
| **Behavioral** | Describes *what* the circuit does using a `process` block. Most abstract. |
| **Dataflow** | Describes *how* data flows using concurrent signal assignments. No process block. |
| **Structural** | Describes the circuit as interconnected sub-components. Closest to actual hardware. |

In this lab, the **Dataflow** style is primarily used for its simplicity.

---

### Basic Data Types

- **`std_logic`** — Standard type for a single-bit signal. Supports 9 values including `'Z'` (High Impedance) and `'U'` (Uninitialized).
- **`std_logic_vector`** — Represents a group of bits (a bus). Example: `std_logic_vector(7 downto 0)` is an 8-bit bus.
- **Signals** — Internal wires declared inside an architecture between the `is` keyword and `begin`.

---

### VHDL Design Cycle

1. **Analysis (Compilation):** Checks source code for syntax errors.
2. **Elaboration:** Flattens the hierarchical design into a netlist and binds entities to architectures.
3. **Simulation:** Applies input stimuli to verify functional behavior over time.
4. **Visual Verification:** GTKWave reads the generated `.vcd` file to display signal timing diagrams.

---

### Simulation Commands (GHDL)

```bash
# 1. Analyze (compile) both files
ghdl -a buffer.vhd buffer_tb.vhd

# 2. Elaborate (link) the testbench
ghdl -e buffer_tb

# 3. Simulate and export waveform
ghdl -r buffer_tb --vcd=simulation.vcd

# 4. Open waveform in GTKWave
gtkwave simulation.vcd
```

---

## Design File

**Filename:** `buffer.vhd`

The design file implements a simple **buffer** circuit — the output directly follows the input. It is modeled using the Dataflow architectural style.

---

## Testbench File

**Filename:** `buffer_tb.vhd`

The testbench has no input or output ports. It instantiates the buffer as the Design Under Test (DUT) and applies a sequence of input stimuli (`'0'`, `'1'`, `'0'`) at 10 ns intervals to verify that the output follows the input correctly.

---

## Simulation File

**Filename:** `simulation.vcd`

The simulation file is generated by GHDL after running the testbench. It is a **Value Change Dump (VCD)** file that records all signal transitions over the simulation time. It is opened in GTKWave for visual verification.
